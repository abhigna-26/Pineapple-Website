Security Practices

Last Updated: June 15, 2026

Security is not an afterthought at Pineapple. We build custom websites and portals with data protection and vulnerability prevention as core tenets of our architectural decisions.

1. Secure Coding Standards

We write code following modern web security practices, design patterns, and OWASP recommendations. Specifically, we focus on:

  • Input Validation and Output Encoding to prevent Cross-Site Scripting (XSS)
  • Secure parameterized database queries to prevent SQL Injection
  • Strict Content Security Policies (CSP) to restrict unauthorized script executions
  • Avoiding insecure direct object references (IDOR) through rigorous permission mapping

2. Data Encryption

Protecting user and client data in transit and at rest is crucial:

  • All data transmitted between browsers and our websites is encrypted using modern TLS (HTTPS) standards.
  • Sensible information such as passwords, database connections, and api tokens are securely stored and encrypted using salt-hashed algorithms (like bcrypt) at rest.

3. Infrastructure & Deployment

We build on secure hosting platforms (such as Netlify, Vercel, or AWS) that offer advanced protection features:

  • Automated web application firewalls (WAF) to mitigate DDoS attacks
  • Secure deployment pipelines with secret environment variables (never committed to Git)
  • Regular dependency audits using automated vulnerability scanner bots (like Dependabot)

4. Access Controls

We implement role-based access control (RBAC) in admin panels, ensuring users only access information and actions necessary for their specific organizational tasks. Session management tokens are stored in secure, HttpOnly, and SameSite cookies.

5. Reporting Vulnerabilities

We encourage developers and security researchers to report any potential security issues. If you notice a vulnerability in any systems we maintain, please report it to us immediately at pineapple.apt26@gmail.com.
